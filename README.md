## 1. Overview

**rltree** is a minimal reinforcement learning pipeline for generating structured SVG XML trees. A lightweight MLP policy network learns to sequentially construct well-formed SVG documents by issuing discrete tree-building actions — adding child nodes or closing the current node — one step at a time.

The pipeline runs end-to-end on CPU in under 10 seconds, producing a syntactically valid SVG string as output.

---

## 2. Motivation

Tree-structured output is a recurring problem in program synthesis, code generation, and document generation. This prototype investigates whether a very small policy network trained with:

1. **Behavioral cloning** on a tiny synthetic dataset, followed by
2. **REINFORCE** fine-tuning against a dense structural reward signal

can learn to produce valid hierarchical XML without a grammar oracle or constrained decoding.

The scope is deliberately narrow: a fixed 5-tag SVG vocabulary, a maximum depth of 3, and a maximum of 6 nodes per tree. Scaling is out of scope for this prototype.

## 3. Architecture

### 3.1 Dataset Generator

**File:** `rltree/dataset.py`

Generates random synthetic SVG strings under strict structural constraints:

- **Vocabulary:** `svg`, `g`, `rect`, `circle`, `path`
- **Max nodes:** 6 (including root)
- **Max depth:** 3 (root = depth 1)
- **Root tag:** always `<svg>`

Nested `<g>` elements are the only container permitted to have children below depth 1. Leaf tags (`rect`, `circle`, `path`) are placed at any depth within limits. The generator uses seeded randomness for reproducibility.

### 3.2 DFS Trajectory Parser

**File:** `rltree/parser.py`

Converts a ground-truth SVG string into a sequence of `(state, action)` pairs via depth-first traversal. At each node entry, an `add_child` action is recorded. At each node exit, a `close_node` action is recorded. The resulting trajectory is used directly as supervised training data for behavioral cloning.

**State tuple:** `(depth, parent_tag_id, current_tag_id, sibling_index, total_siblings)`

**Action encoding:**

| Action Index | Meaning |
|---|---|
| 0 | `add_child(svg)` |
| 1 | `add_child(g)` |
| 2 | `add_child(rect)` |
| 3 | `add_child(circle)` |
| 4 | `add_child(path)` |
| 5 | `close_node` |

### 3.3 MLP Policy Model

**File:** `rltree/model.py`

A small feedforward network (`TreePolicyMlp`) with:

- Two `nn.Embedding` layers (dim=8 each) for `parent_tag_id` and `current_tag_id`
- Three scalar inputs: `depth`, `sibling_index`, `total_siblings`
- Two hidden layers: `Linear(19 → 64)` + `Linear(64 → 64)` with ReLU activations
- Output layer: `Linear(64 → 6)` producing raw logits over the 6-action space

Total parameters: ~9,000. Intentionally minimal to fit the rapid-PoC time budget.

### 3.4 Tree-Building Environment

**File:** `rltree/env.py`

A Gym-style environment (`SvgBuildingEnv`) that maintains a pointer to the currently active `TreeNode`. At each step the agent issues one of the 6 discrete actions. The environment enforces structural validity (depth and node count limits), manages the active-node pointer, and computes the per-step reward.

The environment optionally ingests training SVGs at construction time to extract observed `(parent, child)` tag pairs, used to compute a schema-adherence bonus.

### 3.5 Training Pipeline

**File:** `rltree/train.py`

Two-phase training coordinated by `run_pipeline()`:

**Phase 1 — Behavioral Cloning (BC):**
Supervised training on DFS trajectories extracted from the synthetic dataset. Uses `CrossEntropyLoss` with inverse-frequency class weights to prevent the dominant `close_node` action from collapsing the gradient signal. Runs for up to 50 epochs with early stopping at 95% validation accuracy.

**Phase 2 — REINFORCE Fine-Tuning:**
Policy gradient updates using Monte Carlo returns with γ = 0.99. Returns are standardised per episode. Runs for 300 episodes. The reward signal shapes the agent away from degenerate strategies (e.g., immediately closing an empty root).

## 4. State and Action Space

The observation state is a 5-tuple of integers:

```
(depth, parent_tag_id, current_tag_id, sibling_index, total_siblings)
```

Tag ID mapping:

| Tag | ID |
|---|---|
| `<pad>` | 0 |
| `svg` | 1 |
| `g` | 2 |
| `rect` | 3 |
| `circle` | 4 |
| `path` | 5 |

The action space is discrete with 6 elements (indices 0–5), as described in §3.2.

---

## 5. Reward Function

| Condition | Reward |
|---|---|
| Valid `add_child` action | `+0.1` |
| Parent-child pair seen in training set (schema bonus) | `+1.0` |
| Invalid `add_child` (exceeds depth, exceeds max nodes, or duplicate root) | `−1.0` |
| `close_node` on root with **no children** (empty-tree shortcut) | `−3.0` |
| `close_node` on root with at least one child | `+5.0 + 0.5 × (node_count − 1)` |
| `close_node` on non-root node (return to parent) | `+0.1` |

The scaled completion bonus rewards richer trees over minimal ones, directly incentivising the agent to build multi-node structures.

## 11. Known Limitations

- **Mode diversity:** The policy frequently collapses to a single tag type (e.g., all `<circle>`). This is a known consequence of training a tiny model on a small dataset with no entropy regularisation.
- **No grammar constraint:** The model can attempt invalid actions. Validity is enforced by the environment's reward signal alone, not by constrained decoding.
- **No persistence:** The trained model is not saved to disk. Each `make run` re-trains from scratch.
- **Batch training:** BC training uses full-batch gradient descent, not minibatches. This is acceptable at this dataset size but does not scale.
- **REINFORCE variance:** Policy gradient estimates are high-variance. With only 300 episodes, learning is noisy and not guaranteed to converge to a stable policy.
- **Fixed seed dependency:** Reproducibility depends on setting `random.seed` in the dataset generator. PyTorch operations are not seeded.
